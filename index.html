TCP 3-WAY HANDSHAKE PROCESS

Transmission Control Protocol is designed to provide secure data communication between users in a connection-oriented network. Connection between the client and server is established prior to data been sent. TCP is a standard working with the internet Protocol and uses a single type of data unit known as the TCP segment.This connections are made possible by the 3-way handshake process.This is carried out in the TCP during connection establishment and closure between the two devices.

TCP MECHANISMS

TCP ensures the safe routing of data and uses different mechanisms to ensure this data is received at a high performance and reduced congestion.

The TCP mechanisms can be grouped into three categories, the connection establishment, the transfer of data, and the termination of the connection.

Connection Establishment

While using the 3-way handshake, the SYN flag is set and this begins the request for connection. The connection is ascertained by the sockets of the source and destination. The source sends a SYN segment and the receiver responds SYN/ACKs. There can solely be only one connection established.

Data Transfer

Each of the segments entails the 32-bit sequence as a stream of octets. TCP decides the period to assemble the transmitted segment and also when to release data acquired to the user. Flags are used to control and monitor the flow of the data. An example is the RST flag which is useful when there are delayed duplicated SYN and ACKs

Connection Termination

For a connection to be terminated each of the users would issue a CLOSE primitive. The FIN bit is then set which signals the end of data transmission. An ABORT primitive however, would cause the connection to terminate abruptly.

3-WAY CONNECTION STEPS

For a connection to be established, TCP would use the 3-way handshake process. It begins with the client attempting to connect with a known server. The server would initially bind and listen prior to opening the connection. After the connection is established, the data is transferred and subsequently terminated. This secure method of connection is useful for CPU reservation, and bandwidth of the hosts for efficient communication.

SYN

The client sends a SYN to the server to begin the connection. Also it sets the sequence number of the segment to any random value X.

SYN-ACK

The server responds with a SYN-ACK and acknowledging the sequence. This number is set to be one more than the sequence that was received > X + 1, while the server chooses a separate number Y for the packet.

ACK

The client then send an ACK to the server and sets the sequence number to the value of the received value(X + 1), and the acknowledged number would be set to one more than the sequence received (Y + 1)

TCP HEADER FILE

There are a couple of elements in the header file of the TCP which are useful in carrying out the handshake process.

Sequence Number: This is a random 32-bit number that is assigned to the initial data bit. This number is only required once at the initial connection establishment.

Acknowledgement Number: This is the next sequence number that the sending device expects, and it is set to 1 greater than the sequence number

Buffer Size: This is also known as the window size, and it is the capacity that the data can be received residing in the buffer.

Max segment size: This is the maximum size within the connected devices containing the data segments. If it exceeds this size, then the devices would not be to receive the segments.

SYN flag This refers to the synchronization when there is a connection establishment. When SYN is 1 then it is assumed that the device is requesting a secure connection.

ACK flag: This refers to acknowledgement, and it is response to SYN. When ACK is 1, then the device has gotten the SYN message.

Fin flag: This refers to Finished signifying the transmission of data is complete. The client(sender) and server(receiver) terminate the connections using this flag. When Fin is 1, then the device is requesting to terminate the connection.
